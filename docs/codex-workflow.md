# Codex開発手順

## この文書の目的

この文書は，このプロジェクトや仕様駆動開発に詳しくない人でも，上から順番にCodexへ指示して開発を進められるようにするための手順書である．

各ステップでは，必ず作業前にCodexへ計画を提示させる．人間が確認するまでは，AWSリソース作成，AWS設定変更，インターネット公開，SSH設定変更，ファイアウォール変更，外向き通信許可を行わない．

## 共通ルール

- 作業前に `README.md`，`AGENTS.md`，この文書，関連する仕様書を読む．
- 各ステップごとに作業ブランチを切る．
- 一機能または一作業ごとにコミットする．
- 文書，計画，説明は原則日本語で書く．
- 技術名，コマンド名，ファイル名，ブランチ名は英語のままでよい．
- Python関連作業ではプロジェクト直下の `.venv` を使用する．
- システムPythonへプロジェクト依存を直接インストールしない．
- テストは可能な範囲で先に追加する．
- テストデータは合成データだけを使用する．
- 生ログ，実IP，認証情報，秘密鍵，取得ファイル，マルウェアサンプルをコミットしない．
- このリポジトリは公開リポジトリとして扱い，第三者が読んでも恥ずかしくない文書，コメント，コミットにする．
- AWSではGitHubからcloneしてデプロイする前提のため，動作に必要なテンプレート，手順，スクリプトはGitで管理する．
- AWS環境固有の実値はGitへ入れず，`.env.example` や文書で設定項目だけを示す．
