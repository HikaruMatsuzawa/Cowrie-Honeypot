# プロジェクト概要

## 目的

本プロジェクトでは、AWS Lightsail上に低対話型SSHハニーポットであるCowrieを構築し、インターネットから届くSSHブルートフォースやLinux/IoT機器を狙う接続の挙動を安全に観測する。

目的は攻撃機能を提供することではなく、外部から届く不正アクセス試行を受動的に記録し、ログを分析できる形に整理することである。

## 対象範囲

初期バージョンではSSHを対象とする。

- TCP 22番への接続
- SSHブルートフォース
- 試行されたユーザー名
- 試行されたパスワード
- ログイン成功として扱われたセッション
- 攻撃者が入力したコマンド
- `wget` や `curl` などのダウンロード試行
- `chmod` やシェル実行などの操作
- LinuxサーバーやIoT機器を狙う自動化された接続
- Cowrie JSONログの保存
- JSONログの正規化
- 集計、分類、匿名化、CSV出力

TelnetのTCP 23番は、SSH構成、実送信元IP記録、外向き通信制限を確認した後に追加する観測対象とする。

## 対象外

初期バージョンでは次を対象外とする。

- 本物の脆弱なLinux OSの公開
- 高対話型ハニーポット
- Windowsマルウェアの実行
- Windows RDPの模擬
- SMBハニーポット
- 攻撃者による任意コード実行
- 攻撃者による外部ホストへの通信
- 取得したマルウェアの実行
- 取得したマルウェアの第三者への提供
- 攻撃者への反撃
- ポートスキャン
- 脆弱性スキャン
- 外部ホストへの攻撃試験
- 自動デプロイ
- Kubernetes
- ECS
- 公開Webダッシュボード

## 成功条件

本プロジェクトの成功条件は、単にCowrieが起動することではない。次のすべてを満たすことを成功条件とする。

- 外部からのSSH接続をCowrieで観測できる。
- 本物の管理用OpenSSHがインターネット全体へ公開されていない。
- Cowrie本体から外部へ攻撃通信を発生させない。
- ログを安全に保存できる。
- ログを再現可能な方法で分析できる。
- 生ログや秘密情報を公開しない。
- 異常時に安全に停止できる。
- Lightsailでのデプロイ、停止、復旧、監視の手順が文書化されている。
- Codexの変更を人間がレビューできる。
- テストと受け入れ条件によって品質を確認できる。
