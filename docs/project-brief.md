# プロジェクト概要

## 目的

本プロジェクトでは，AWS上に低対話型SSHハニーポットであるCowrieを構築し，インターネット上のSSHブルートフォース，LinuxサーバやIoT機器を狙うボットの挙動を安全に観測する．

本物の脆弱なLinux環境を攻撃者へ提供するのではなく，Cowrieが提供する模擬SSH環境を使用する．目的は，攻撃者へ攻撃機能を提供することではなく，外部から届く不正アクセスを受動的に記録し，ログを分析することである．

## 対象範囲

初期バージョンではSSHを対象とする．

- TCP 22番への接続
- SSHブルートフォース
- 試行されたユーザー名
- 試行されたパスワード
- ログイン成功として扱われたセッション
- 攻撃者が入力したコマンド
- `wget` や `curl` などのダウンロード試行
- `chmod` やシェル実行などの操作
- Linuxサーバを狙う自動化ボット
- 組み込みLinuxやIoT機器を狙うボット

TelnetのTCP 23番は，初期バージョンでは公開しない．SSH構成が安定し，安全性を確認した後の拡張候補とする．

## 対象外

初期バージョンでは，次の機能を対象外とする．

- 本物の脆弱なLinux OSの公開
- 高対話型ハニーポット
- Windowsマルウェアの実行
- Windows RDPの模擬
- SMBハニーポット
- 攻撃者による任意コード実行
- 攻撃者による外部ホストへの通信
- 取得したマルウェアの実行
- 取得したマルウェアの第三者への提供
- 攻撃元への反撃
- ポートスキャン
- 脆弱性スキャン
- 外部ホストへの攻撃試験
- 自動攻撃機能
- ボットネットへの参加
- 複数種類のハニーポットを含むT-Pot構成
- Kubernetes
- ECS
- 自動デプロイ
- 公開Webダッシュボード

## 成功条件

本プロジェクトの成功条件は，単にCowrieが起動することではない．次のすべてを満たすことを成功条件とする．

- 外部からのSSH攻撃を観測できる
- 本物の管理用SSHが公開されていない
- Cowrieから外部へ攻撃通信を発生させない
- ログを安全に保存できる
- ログを再現可能な方法で分析できる
- 秘密情報や生ログを公開しない
- 異常時に安全に停止できる
- 構成と判断理由が文書化されている
- Codexの変更を人間がレビューできる
- テストと受け入れ条件によって品質を確認できる
